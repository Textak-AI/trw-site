# The Rail Way Site Editor — Ella's Quick Guide

Welcome! This guide walks you through editing small things on therailway.us — text changes, date fixes, team bio tweaks, FAQ updates. For anything bigger (a new page, a new article, a new team member, layout changes), Kyle is still your person.

---

## Logging in

1. Go to **https://therailway.us/editor**
2. Enter your email and password (Kyle will send these separately)
3. Click **Sign In**

If you see a "Not authenticated" message after logging in, refresh the page.

---

## Finding what to edit

The left sidebar lists every page on the site:
- **Home Page** — the landing page with hero, stats, team, etc.
- **Speak Up Culture** — the Speak Up Culture explainer
- **Our Process** — the four-stage process page
- **About Pauline** — Pauline's bio page
- **Insights** — the articles index page
- **Class II & III** — the smaller railroads page
- **Contact** — the contact form page
- **Legacies** — the honoring/in-memoriam page
- **Events** — Pauline's speaking engagements

Click a page on the left. The middle area shows all the editable sections for that page.

---

## Editing text

1. Click a **section** (like "Hero / Top Banner") to expand it
2. Edit any field you want
3. When you're done, click **Publish Changes** at the top right

That's it. Your edits will be live on the site in about **1–2 minutes**.

---

## Editing lists (team members, FAQ items, stats, etc.)

When a section has a list of items (team members, FAQ questions, event dates, etc.), you'll see each item as a collapsible card.

- **Click an item** to expand it and see its fields
- **↑ Up / ↓ Down** — reorder items
- **Remove** — delete an item (you'll get a confirmation)
- **+ Add another** — at the bottom of each list, adds a new blank item

## If something breaks

**If the editor won't load:** refresh your browser.

**If Publish Changes fails:** the banner at the top will tell you the error. Screenshot it and send to Kyle.

**If you published something by accident:** don't panic. Undo the change in the editor (re-edit the field back to what it was, or use Discard if you haven't left the page yet) and click Publish Changes again. Every publish creates a version history in the code repo that Kyle can roll back if needed.

**If the live site doesn't update after 5 minutes:** ping Kyle with the page you edited. Sometimes the build queue is slow.

---

## A few things worth knowing

- **Your changes are saved as drafts in your browser until you click Publish.** If you close the tab without publishing, your edits are lost.
- **"Discard" resets everything you've edited this session** — use it if you want to start over.
- **You'll get a warning if you try to leave with unsaved changes.** This is on purpose.
- The URL fields (like Button URL) should always start with `/` for internal links (e.g. `/contact`) or `https://` for external links (e.g. `https://www.railwayage.com/...`).
